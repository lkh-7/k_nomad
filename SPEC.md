## Phase 2: 좋아요/싫어요 인터랙션 개선

### 오버뷰

현재 좋아요/싫어요 버튼은 한 번만 클릭 가능하고 색상 변경이 없는 상태입니다. 이를 개선하여 좋아요 클릭 시 아이콘이 파란색(또는 초록색)으로 활성화되고, 싫어요 클릭 시 아이콘이 빨간색으로 활성화되도록 합니다. 두 버튼은 상호 배타적으로 동작합니다(좋아요 상태에서 싫어요 클릭 시 좋아요 해제 + 싫어요 활성화). 이미 활성화된 버튼을 다시 클릭하면 토글(해제)됩니다. 숫자도 실시간으로 반영됩니다.

**영향 파일:**
- `frontend/components/LikeButtons.tsx` — 전면 개선
- `frontend/components/CityCard.tsx` — LikeButtons에 초기값(likes/dislikes) 전달 확인

### 수정/개선 체크리스트

- [ ] `LikeButtons.tsx`에 `liked` 상태 (`'like' | 'dislike' | null`) 관리 추가
- [ ] 좋아요 버튼 클릭 시 동작 구현:
  - `liked === 'like'` 이면 → `liked = null` (토글 해제), 좋아요 카운트 -1
  - `liked === 'dislike'` 이면 → `liked = 'like'`, 좋아요 +1, 싫어요 -1
  - `liked === null` 이면 → `liked = 'like'`, 좋아요 +1
- [ ] 싫어요 버튼 클릭 시 동작 구현:
  - `liked === 'dislike'` 이면 → `liked = null` (토글 해제), 싫어요 카운트 -1
  - `liked === 'like'` 이면 → `liked = 'dislike'`, 싫어요 +1, 좋아요 -1
  - `liked === null` 이면 → `liked = 'dislike'`, 싫어요 +1
- [ ] 좋아요 버튼 활성화 스타일 적용 (예: `liked === 'like'`일 때 아이콘/버튼 색상 파란색 또는 초록색)
- [ ] 싫어요 버튼 활성화 스타일 적용 (예: `liked === 'dislike'`일 때 아이콘/버튼 색상 빨간색)
- [ ] 비활성화 상태 버튼 기본 스타일 유지 (회색 계열)
- [ ] 좋아요/싫어요 숫자 카운트가 버튼 옆에 실시간으로 표시되는지 확인
- [ ] 로컬 상태로만 관리 (DB 연동 없음)

### 검증/확인 체크리스트

- [ ] 좋아요 버튼 클릭 시 아이콘 색상이 즉시 변경되는지 확인
- [ ] 싫어요 버튼 클릭 시 아이콘 색상이 즉시 변경되는지 확인
- [ ] 좋아요 클릭 후 싫어요 클릭 시 좋아요가 해제되고 싫어요가 활성화되는지 확인
- [ ] 이미 활성화된 좋아요 버튼을 다시 클릭하면 해제(카운트 원복)되는지 확인
- [ ] 이미 활성화된 싫어요 버튼을 다시 클릭하면 해제(카운트 원복)되는지 확인
- [ ] 좋아요 숫자가 클릭에 따라 +1/-1로 변경되는지 확인
- [ ] 싫어요 숫자가 클릭에 따라 +1/-1로 변경되는지 확인
- [ ] 여러 도시 카드의 좋아요/싫어요가 서로 독립적으로 동작하는지 확인

## Phase 5: 카드 UI 개편 + 도시 리스트 타이틀 및 정렬 변경

### 오버뷰

도시 카드에 Phase 3에서 추가한 4개 필드(예산/지역/환경/최고계절)를 Key-Value 형태로 시각적으로 표시합니다. 기존 점수 바와 태그 칩 영역을 이 정보로 대체합니다. 아울러 섹션 타이틀을 'Top추천도시'에서 '도시리스트'로 변경하고, 기본 정렬을 좋아요 순으로 변경합니다.

**영향 파일:**
- `frontend/components/CityCard.tsx` — Key-Value 정보 표시로 카드 본문 개편
- `frontend/app/page.tsx` 또는 `CityGrid.tsx` — 섹션 타이틀 변경
- `frontend/components/FilterBar.tsx` 또는 `CityGrid.tsx` — 기본 정렬 변경

### 수정/개선 체크리스트

- [ ] `CityCard.tsx` 본문 영역에 Key-Value 블록 구현:
  - `월 예산` → `{city.budget}`
  - `지역` → `{city.regionCategory}`
  - `환경` → `{city.environment}`
  - `추천 계절` → `{city.bestSeason}`
- [ ] Key-Value 스타일 정의: Key는 회색 레이블, Value는 흰색 또는 강조색으로 명확히 구분
- [ ] 기존 태그 칩(`바다`, `카공`, `저렴` 등) 표시 제거
- [ ] 기존 점수 바(ScoreBar)가 Phase 1에서 제거되었는지 이중 확인
- [ ] `CityCard.tsx`에서 `nomadScore`, `scores` 관련 렌더링 코드 제거
- [ ] 섹션 타이틀 `'Top 추천 도시'` → `'도시리스트'` 변경 (해당 텍스트가 있는 파일 확인 후 수정)
- [ ] 기본 정렬 값 변경: `sortBy` 초기값을 `'nomadScore'`에서 `'likes'`로 변경
- [ ] 정렬 드롭다운에서 `'좋아요순'`이 기본으로 선택된 상태로 표시되도록 확인
- [ ] 카드 레이아웃이 Key-Value 추가 후에도 반응형으로 정상 표시되는지 확인

### 검증/확인 체크리스트

- [ ] 도시 카드에 `월 예산`, `지역`, `환경`, `추천 계절` 4개 항목이 Key-Value로 표시되는지 확인
- [ ] 각 카드의 Key-Value 값이 Phase 3에서 입력한 데이터와 일치하는지 확인
- [ ] 섹션 타이틀이 `'도시리스트'`로 표시되는지 확인 (`'Top추천도시'` 텍스트 없음)
- [ ] 페이지 최초 로드 시 도시 목록이 좋아요 수 내림차순으로 정렬되어 있는지 확인
- [ ] 좋아요 수가 가장 많은 도시가 첫 번째에 위치하는지 확인
- [ ] 기존 태그 칩(`바다`, `카공`, `저렴` 등)이 카드에 표시되지 않는지 확인
- [ ] 카드 UI가 모바일/태블릿/데스크톱 각 화면에서 깨지지 않는지 확인
- [ ] 필터 + 좋아요 정렬이 함께 동작하는지 확인 (필터 적용 후에도 좋아요 순 정렬 유지)
- [ ] `npm run build` 빌드 에러 없음 확인
