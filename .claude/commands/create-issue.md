# 역할

너는 GitHub 이슈를 잘 작성하는 시니어 개발자야. 입력된 내용을 분석해서 명확하고 실행 가능한 GitHub 이슈를 생성해줘.

# 생성할 이슈

$ARGUMENTS

---

# 작업 흐름

## 1단계: 이슈 분류

입력 내용을 보고 아래 중 하나로 분류해:
- `feature` — 새 기능 추가
- `bug` — 버그 수정
- `docs` — 문서/주석 작업
- `chore` — 리팩토링, 설정, 의존성 등

## 2단계: 이슈 구성 요소 작성

아래 구조로 이슈 내용을 구성해:

**제목 규칙:**
- `[타입] 동사 + 목적어` 형식
- 예: `[feature] 도시 상세 페이지 추가`, `[bug] 좋아요 카운트 새로고침 후 초기화 안됨`
- 50자 이내, 핵심만

**본문 구조:**
```
## 배경
[왜 이 이슈가 필요한지 1~3줄]

## 작업 내용
[무엇을 해야 하는지 구체적으로]

## 수락 기준
- [ ] [확인 가능한 조건 1]
- [ ] [확인 가능한 조건 2]
- [ ] [확인 가능한 조건 3]

## 기술 참고
[관련 파일, 참고할 구현 방식, 주의사항 - 없으면 이 섹션 생략]
```

## 3단계: gh CLI로 이슈 생성

아래 명령어로 실제 GitHub 이슈를 생성해:

```bash
gh issue create \
  --title "[제목]" \
  --body "[본문]" \
  --label "[label]"
```

생성 완료 후 이슈 URL을 출력해줘.

---

## 주의사항
- 제목은 간결하게, 본문은 구체적으로
- 수락 기준은 반드시 체크박스(`- [ ]`) 형식으로
- 기술적으로 불명확한 부분은 본문에 `[확인 필요: 내용]`으로 표시
- label이 저장소에 없을 수 있으니 없으면 `--label` 옵션 제외하고 실행
